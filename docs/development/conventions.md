# Conventions

Code style and repo conventions. The executable source of truth is
`biome.json`, `tsconfig.json`, and the pre-push hook ŌĆö this page explains the
*why* and the non-obvious rules.

## Formatting & linting

- **Biome** is the formatter and linter (`pnpm lint` = `biome check .`).
  Config in `biome.json`.
- **Pre-push hook** (`.husky/pre-push`) runs `biome check` on changed
  `*.{ts,tsx,js,mjs,jsx}` files only ŌĆö not the whole codebase, to avoid tripping
  on pre-existing lint debt in untouched files. It also scans tracked files for
  common secret patterns and aborts the push on a match.
- Re-stage modified files and retry if the hook reformats.

## TypeScript

- `tsconfig.json` ŌĆö strict mode; `baseUrl` was removed (deprecated) and `tsc` is
  clean. `pnpm typecheck` runs `tsc --noEmit`.
- `cloudflare-env.d.ts` is generated by `pnpm cf:typegen` and is gitignored ŌĆö
  do not commit it.

## Database

- **NO ORM.** Raw SQL via `@libsql/client`. Schema in `src/db/schema.sql`;
  applied wholesale by `pnpm db:migrate`. Do not hand-edit the Turso table ŌĆö
  the migrate path is the only safe one.
- **Tags** stored as JSON arrays in `user_repos.tags` text column.
- **Embedding dimension contract** ŌĆö `EMBEDDING_DIM=768` pinned across
  `src/lib/embeddings.ts`, `src/db/schema.sql`, `src/db/migrate.ts`. Change all
  three together. See
  [../architecture/decisions/0006-embedding-dimension-contract.md](../architecture/decisions/0006-embedding-dimension-contract.md).

## Client state & data fetching

- **nuqs** for all filter/sort state in URL params (shareable links). Search,
  sort, language, list, tag params all via nuqs.
- **SWR** for all client data via hooks in `src/hooks/`. No React Query.
- **Virtualized grid** via `@tanstack/react-virtual` for 1000+ repos.

## Auth

- **NextAuth v5 beta** (not v4 stable) ŌĆö session shape differs. Access token
  threaded through JWT/session callbacks for GitHub API sync.
- `trustHost: true` is hardcoded in `src/lib/auth.ts` ŌĆö the env-driven
  `AUTH_TRUST_HOST` was unreliable in the Worker. Both `AUTH_URL` and
  `NEXTAUTH_URL` must be set in `wrangler.jsonc` vars. See
  [../knowledge/learnings.md](../knowledge/learnings.md).

## Cloudflare Workers build

- `src/db/index.ts` imports `@libsql/client/web` (not `@libsql/client`) and
  lazy-inits via a Proxy. Using the default import will re-break the Worker.
- The CF build uses `--webpack` (not Turbopack). See
  [../architecture/decisions/0003-opennext-libsql-bundling.md](../architecture/decisions/0003-opennext-libsql-bundling.md).
- `open-next.config.ts` uses `staticAssetsIncrementalCache` so prerendered HTML
  with Beasties-inlined critical CSS is served from the assets binding.

## Generated files ŌĆö do not hand-edit

- `agent-edge.mjs` ŌĆö regenerated by the fleet agent-edge generator.
- `worker.mjs` ŌĆö OpenNext-generated Worker entry.
- `cloudflare-env.d.ts` ŌĆö generated by `wrangler types`.
- `data/fleet-projects.generated.json` ŌĆö generated by
  `pnpm fleet:extract-projects`.

## Agent skills / plugins ŌĆö do not modify

Do not modify, move, rename, or delete agent skills, plugins, or agent-profile
directories (`.claude/`, `.codex/`, `.symphony/`, `.clawpatch/`, any `SKILL.md`).
They are tooling, not product code.
