# Architecture Overview

Starboard is a Next.js 16 (App Router, React 19) application deployed to a
Cloudflare Worker named `starboard` via `@opennextjs/cloudflare`. Data lives in
Turso (libSQL) with raw SQL — no ORM. Semantic search uses the shared fleet
`knowledgebase` Worker (RAG) when configured, with a Turso `F32_BLOB(768)`
vector index retained for non-RAG features (similar repos, discover,
recommendations).

## Shape

```
                ┌──────────────────── Cloudflare Worker (starboard) ────────────────────┐
                │                                                                       │
  browser ──────► Next.js App Router (OpenNext bundle, worker.mjs entry)                 │
                │   ├── /stars               dashboard (filters, search, virtual grid)   │
                │   ├── /explore/[...slug]   repo detail (comments, votes, similar)      │
                │   ├── /discover            seeded popular repos + tool facets          │
                │   ├── /projects, /projects/[slug]   fleet-aware recommendations        │
                │   ├── /lists/[slug]        public shared list (SSR)                    │
                │   ├── /radar               maintainer/release signals                  │
                │   ├── /reports/[slug]      shareable insight reports                   │
                │   ├── /stack-builder, /tools, /about, /privacy, /terms                 │
                │   ├── /api/*               route handlers (stars, repos, lists, ...)   │
                │   └── agent-edge.mjs       /llms.txt, /index.md, /api/ai, /robots.txt  │
                │                                                                       │
                │   Bindings: ASSETS (.open-next/assets), AI (Workers AI),               │
                │             RAG_SERVICE (service binding → knowledgebase Worker)       │
                └──────────────────────────────────┬────────────────────────────────────┘
                                                   │
            ┌──────────────────────────────────────┼─────────────────────────────────┐
            ▼                                      ▼                                 ▼
       Turso (libSQL)                  Cloudflare Workers AI           knowledgebase Worker
       users, repos, user_repos,       @cf/baai/bge-base-en-v1.5       (shared RAG, optional)
```

## Key files

- `worker.mjs` — OpenNext-generated Worker entry; wraps the Next.js server with
  a global try/catch error handler and the agent-edge handler.
- `agent-edge.mjs` — generated agent-indexing handler (`/llms.txt`,
  `/llms-full.txt`, `/index.md`, `/api/ai`, `/robots.txt`). Regenerated by the
  fleet agent-edge generator; do not hand-edit.
- `next.config.ts` — `transpilePackages: ["@libsql/client"]` + OpenNext dev init
  for local binding access. See
  [decisions/0003-opennext-libsql-bundling.md](decisions/0003-opennext-libsql-bundling.md).
- `open-next.config.ts` — `staticAssetsIncrementalCache` so prerendered HTML
  (with Beasties-inlined critical CSS) is served from the assets binding.
- `wrangler.jsonc` — Worker config: `main = worker.mjs`, `ASSETS` + `AI` +
  `RAG_SERVICE` bindings, `nodejs_compat`, custom domain
  `starboard.codevetter.com`.
- `src/db/index.ts` — Turso client singleton. Imports `@libsql/client/web` and
  lazy-inits via a Proxy (env not available at module load in Workers).
- `src/db/schema.sql` — full schema (6 core tables + vector + snapshot + tool +
  AI metadata + alerts + reports + seed_cursor tables, 7 indexes + vector
  index). Applied wholesale by `pnpm db:migrate`.
- `src/db/migrate.ts` — migration runner; includes `ensureEmbeddingDimension()`
  self-heal. See
  [decisions/0006-embedding-dimension-contract.md](decisions/0006-embedding-dimension-contract.md).
- `src/lib/embeddings.ts` — dual-path embedding (Workers AI binding preferred,
  HTTP gateway fallback). `EMBEDDING_DIM=768` pinned.
- `src/lib/github.ts` — GitHub star sync with ETag caching.
- `src/lib/github-lists.ts` — GitHub star-list HTML scraping (no official API).
- `src/lib/search.ts` — `rrfFuse()` and `cosineSimilarity()` utilities.
- `src/lib/knowledgebase.ts` — shared RAG Worker client (relevance search +
  ingest).
- `src/lib/fleet-projects.ts` — My Projects recommendation scorer against
  `data/fleet-projects.generated.json`.
- `src/lib/recommendation-eval.ts` — fixture-backed ranking eval harness.
- `src/lib/auth.ts` — NextAuth v5 config; `trustHost: true` hardcoded.

## Architecture notes (quick reference)

- **NO ORM** — raw SQL via `@libsql/client`. Schema in `src/db/schema.sql`;
  apply with `pnpm db:migrate`.
- **nuqs** — all filter/sort state in URL params (shareable links).
- **SWR** — all client data through SWR hooks. No React Query.
- **Tags** stored as JSON arrays in `user_repos.tags` text column.
- **ETag caching** for GitHub star sync.
- **Facets** (language/list/tag counts) computed server-side in `GET /api/stars`.
- **Virtualized grid** via `@tanstack/react-virtual`.
- **NextAuth v5 beta** — session shape differs from v4; access token threaded
  for GitHub API sync.
- **SaaS Maker** — feedback, testimonials, changelog widgets integrated.
  PostHog is the analytics path (SaaS Maker analytics removed).
- Pre-push hook runs Biome lint on changed files.
